This tutorial explores which social apps people tend to use together. Rather than jumping to a heavy market-basket library, I use a transparent, lightweight approach to generate association rules and explain the result metrics in plain language. If you’re new to association rules, this is a great “first principles” walkthrough you can reuse on any binary-indicator dataset (apps, features, products).

What you’ll learn: how to (1) clean binary data for rules; (2) compute support, confidence, and lift; (3) enumerate frequent itemsets; (4) rank rules; and (5) translate findings into marketing actions.

Purpose / Objective

Identify pairs of social apps that share audience usage to inform cross-promotion, bundled offers, and partner campaigns. The objective is a short, lift-ranked list of co-usage rules that marketers can act on.

Methodology

Why co-usage?

Co-usage reveals audience overlap. If two apps are frequently used by the same people, they’re natural candidates for sequenced messaging and partnership activations.

Data & dictionary

Scope: 100 respondents; each row is a person. Columns are 1/0 indicators for regular app use.

  • User ID — dropped (not used in modeling)
  • Apps — e.g., Instagram, Snapchat, Facebook, Twitter, TikTok, YouTube (1 = uses)

Assumptions: Self-reported regular use; small sample illustrative.

Step 1 — Prep & sanity checks

Remove User ID, coerce app columns from 1/0 to TRUE/FALSE, and verify no all-zero or duplicate columns.

Tip: Impute missing with 0 unless “don’t know” is a valid category.

Step 2 — Support, confidence, lift

Support: how common the combo is (e.g., 0.30 means 30% of users).
Confidence: reliability of a rule (share of antecedent users who also use the consequent).
Lift: how much more likely the consequent is vs baseline (1.277 ⇒ 27.7% more likely).

Step 3 — Build & rank rules

Enumerate candidate itemsets; keep those with support ≥ 0.20; create rules with confidence ≥ 0.60; rank by lift (tie-break by confidence, then support).

Top rules by lift (higher is stronger than baseline co-usage).

Key Takeaways

Findings — Top pairs & interpretation

Rules meeting thresholds: 8

Top rule: {Snapchat} ⇒ {Instagram}

Support: 0.30 (30 users)

Confidence: 0.638 (63.8% of Snapchat users also use Instagram)

Lift: 1.277 → Snapchat users are 27.7% more likely than average to use Instagram.

This pairing indicates a strong visual-sharing audience overlap. Cross-platform creative referencing both apps should perform well.

Small-n caution: With 100 respondents, treat lift as directional. Validate on a larger panel or usage logs before scaling spend.

Actions — How to use this

  • Cross-promotion: Bundle offers (“follow on Snapchat & Instagram for early drops”).
  • Audience extension: Start with Snapchat audiences, retarget on Instagram; test look-alikes.
  • Creative sequencing: Tease on Snapchat, deepen engagement on Instagram.
  • Partnerships: Prioritize pairs with both high lift and sufficient support.

Appendix — Minimal R (no heavy packages)

# df: 1/0 indicators for each app (exclude "User ID" beforehand)
X <- df; X[] <- lapply(X, function(x) as.logical(as.integer(x)))
min_supp <- 0.20; min_conf <- 0.60

supp <- function(items, M=X){
  if (length(items)==1) mean(M[[items]])
  else mean(rowSums(M[, items, drop=FALSE]) == length(items))
}

# Frequent itemsets
vars <- names(X); freq <- list()
for (k in 1:length(vars)){
  for (items in combn(vars, k, simplify=FALSE)){
    s <- supp(items); if (s >= min_supp)
      freq[[paste(items, collapse=" & ")]] <- list(items=items, support=s)
  }
}

# Rules + metrics
rules <- do.call(rbind, lapply(freq, function(f){
  I <- f$items; s_xy <- f$support
  if (length(I) < 2) return(NULL)
  out <- list(); j <- 1L
  for (r in 1:(length(I)-1)) for (A in combn(I, r, simplify=FALSE)){
    B <- setdiff(I, A); conf <- s_xy / supp(A)
    if (conf >= min_conf){
      lift <- conf / supp(B)
      out[[j]] <- data.frame(
        Antecedent = paste(A, collapse=", "),
        Consequent = paste(B, collapse=", "),
        Support = s_xy, Confidence = conf, Lift = lift
      ); j <- j + 1L
    }
  }
  if (length(out)) do.call(rbind, out)
}))
rules <- rules[order(-rules$Lift, -rules$Confidence, -rules$Support), ]
head(rules)
