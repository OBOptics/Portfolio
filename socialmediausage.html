This tutorial explores which social apps people tend to use together. Rather than jumping to a heavy market-basket library, I use a transparent, lightweight approach to generate association rules and explain the result metrics in plain language. If you’re new to association rules, this is a great “first principles” walkthrough you can reuse on any binary-indicator dataset (apps, features, products).

What you’ll learn: how to (1) clean binary data for rules; (2) compute support, confidence, and lift; (3) enumerate frequent itemsets; (4) rank rules; and (5) translate findings into marketing actions.

Why co-usage matters

Co-usage reveals audience overlap. If two apps are frequently used by the same people, they’re natural candidates for cross-promotion, bundled offers, or partner campaigns. Marketing teams can also build targeted sequences (e.g., show App B creatives to App A users) to boost adoption where lift is strongest.

Data & dictionary

Scope: 100 respondents, each row is a person. Columns are 1/0 binary indicators indicating whether the person regularly uses each app.

  • User ID — dropped (not used in modeling)
  • App columns — e.g., Instagram, Snapchat, Facebook, Twitter, TikTok, YouTube… (1 = uses, 0 = doesn’t)

Assumptions: Users answered truthfully about regular use; sample is small and illustrative.

Step 1 — Prep & sanity checks

I removed User ID and coerced each app column from 1/0 to TRUE/FALSE. I also verified there were no all-zero columns (which would never form a frequent itemset) and no fully collinear duplicates.

Tip: If you have missing values, impute with 0 (not used) unless the survey explicitly allows “don’t know.”

Step 2 — Support, confidence, lift (in plain English)

Support measures how common a combination is across the whole sample. If 30 out of 100 people use both Snapchat and Instagram, support({Snapchat, Instagram}) = 0.30.

Confidence measures reliability of a rule. For {Snapchat} ⇒ {Instagram}, it’s the share of Snapchat users who also use Instagram. If 63.8% of Snapchat users also use Instagram, confidence = 0.638.

Lift tells you how much more likely the consequent is within the antecedent group vs the general population. A lift of 1.277 means “27.7% more likely than baseline.” This is often the most useful business lens.

Step 3 — Build & rank rules (transparent approach)

With a modest number of apps, simple enumeration is fast and keeps the logic obvious. I generated all candidate itemsets, filtered those with support ≥ 0.20, and then created rules with confidence ≥ 0.60. Finally, I ranked by lift (tie-break by confidence, then support).

Top rules by lift (higher is stronger than baseline co-usage).

Findings — Top pairs & interpretation

Rules meeting thresholds: 8

Top rule: {Snapchat} ⇒ {Instagram}

Support: 0.30 (30 users)

Confidence: 0.638 (63.8% of Snapchat users also use Instagram)

Lift: 1.277 → Snapchat users are 27.7% more likely than the average user to use Instagram.

This pattern suggests a high-overlap audience with strong visual-sharing behavior. If you’re planning cross-promotions, this pair is a safe bet: creative that references both platforms should resonate and convert.

Small-n caution: With 100 respondents, treat lift as directional. Validate on a larger panel or a recent usage log (if available) before scaling spend.

Actions — How to use this

  • Cross-promotion: Bundle offers (e.g., “follow us on both Snapchat & Instagram for early sneak peeks”).
  • Audience extension: Start with Snapchat audiences, retarget on Instagram; test look-alikes.
  • Creative sequencing: Tease short/ephemeral content on Snapchat, drive to richer Instagram posts.
  • Partnerships: Align influencer campaigns where lift and support are both high.

Appendix — Minimal R (no heavy packages)

# df: 1/0 indicators for each app (exclude "User ID" beforehand)
X <- df; X[] <- lapply(X, function(x) as.logical(as.integer(x)))
min_supp <- 0.20; min_conf <- 0.60

supp <- function(items, M=X){
  if (length(items)==1) mean(M[[items]])
  else mean(rowSums(M[, items, drop=FALSE]) == length(items))
}

# Frequent itemsets
vars <- names(X); freq <- list()
for (k in 1:length(vars)){
  for (items in combn(vars, k, simplify=FALSE)){
    s <- supp(items); if (s >= min_supp)
      freq[[paste(items, collapse=" & ")]] <- list(items=items, support=s)
  }
}

# Rules + metrics
rules <- do.call(rbind, lapply(freq, function(f){
  I <- f$items; s_xy <- f$support
  if (length(I) < 2) return(NULL)
  out <- list(); j <- 1L
  for (r in 1:(length(I)-1)) for (A in combn(I, r, simplify=FALSE)){
    B <- setdiff(I, A); conf <- s_xy / supp(A)
    if (conf >= min_conf){
      lift <- conf / supp(B)
      out[[j]] <- data.frame(
        Antecedent = paste(A, collapse=", "),
        Consequent = paste(B, collapse=", "),
        Support = s_xy, Confidence = conf, Lift = lift
      ); j <- j + 1L
    }
  }
  if (length(out)) do.call(rbind, out)
}))
rules <- rules[order(-rules$Lift, -rules$Confidence, -rules$Support), ]
head(rules)
